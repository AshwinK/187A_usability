Recognition Rather than Recall


"Minimize the user's memory load by making objects, actions, and options visible. The user should not have to remember information from one part of the dialogue to another. Instructions for use of the system should be visible or easily retrievable whenever appropriate." - Nielsen


Every single page on the website has a navigation bar, but some have more than others and some have less than others. There isn't really any way to tell where you've navigated to within the site, except if there's a header. In the picture provided, the header clearly states "About Us," but there are no breadcrumb mechanisms for backward navigating. You don't really have to recall anything, as you can just go back to the home page and start navigation again - although, that process is a bit annoying. The only problem with the headers and nav-bar titles is that there are repetitive pages, an example being: there are two about us pages, one for marketing and one for the academics. If you can't see the navigation path, you're not going to know which one you're at, how you got there, or how to get back.
